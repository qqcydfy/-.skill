# 仿真项目周报生成器

为仿真分析类项目生成专业周报，输出为Word文档格式。周报内容经过精心组织，可直接用于最终技术报告。

## 使用场景

适用于以下类型的项目周报：
- 热学仿真（TEC、VCSEL芯片温度场分析等）
- 光学仿真（光束传播、光学系统设计等）
- 结构力学仿真（应力分析、模态分析等）
- 电磁场仿真（天线设计、电磁兼容等）
- 流体仿真（CFD分析等）
- 多物理场耦合仿真

## 工作流程

### 第一步：收集本周工作内容

询问用户（或从对话中提取）以下信息：

1. **基本信息**
   - 周报时间范围（如：4.24-4.30）
   - 作者姓名
   - 项目名称（可选）

2. **工作内容**（按模块组织）
   - 每个模块包含：
     - 模块标题（如：①VCSEL芯片布局）
     - 具体工作描述
     - 使用的方法/技术
     - 关键参数（如适用）

3. **仿真结果**
   - 结果图表（图片文件路径或目录）
   - 结果描述与分析
   - 关键发现/结论

4. **总结与展望**（可选）
   - 本周工作总结
   - 下周计划
   - 遇到的问题与解决方案

### 第二步：内容组织与写作

按以下结构组织周报内容。写作时注意：内容要专业、准确、可直接用于技术报告。

```
[周报标题] [作者姓名]

进展：
[一段话概述本周主要工作]

①[模块1标题]
[模块1详细内容，包含方法描述、参数说明、结果分析]

图 X [图片说明]
备注：[数据来源、参数来源等补充说明]

②[模块2标题]
...

③[模块3标题]
...

3）总结
[总结性陈述，提炼关键发现]
```

### 第三步：图片处理

#### 自动插入图片

如果用户提供了图片目录，按以下逻辑自动匹配和插入：

1. **扫描图片目录**：读取目录中的所有图片文件（png, jpg, jpeg, bmp, tiff）

2. **智能匹配**：根据以下规则将图片与内容关联
   - 按文件名中的数字顺序匹配文中"图 X"的序号
   - 按文件名关键词匹配（如包含"温度"、"温差"等关键词）
   - 如果无法自动匹配，在文中插入占位符：`[请在此处插入图片：描述]`

3. **插入格式**：
   - 图片居中显示
   - 图片下方添加说明文字（Caption样式）
   - 图片宽度适配页面（建议宽度15cm）

#### 手动插入占位符

对于无法自动匹配的图片，在文档中标注：
```
[图片占位符：图 X - 简要描述]
位置：在"[图 X 说明文字]"之后
建议尺寸：宽度15cm，高度按比例
```

### 第四步：生成Word文档

使用 `scripts/generate_docx.py` 脚本生成最终文档。

```bash
python skills/weekly-report/scripts/generate_docx.py \
  --config weekly_report_config.json \
  --output "周报_姓名_日期.docx"
```

## 写作指南

### 专业性要求

1. **术语准确**：使用领域标准术语，首次出现时可简要解释
2. **数据精确**：仿真结果保留适当精度（如温度保留1位小数：0.012K）
3. **逻辑清晰**：按照"方法→结果→分析"的顺序组织内容
4. **可追溯性**：注明参数来源（如"来源于VCSEL芯片datasheet典型值"）

### 内容风格

- 客观陈述，避免主观评价
- 使用第三人称或无主语句式
- 结果分析要基于数据，说明趋势和原因
- 总结部分提炼关键发现，不过度推论

### 直接用于技术报告的要点

周报内容要做到：
1. **独立完整**：不依赖周报外的上下文
2. **数据准确**：所有数值、图表都有明确来源
3. **逻辑自洽**：结论与分析对应，分析与数据对应
4. **格式规范**：符合技术文档写作规范

## 输出格式

### Word文档结构

```
标题：[时间范围]周报 [作者姓名]  （Normal样式，居中）

正文：
- 进展概述（Normal样式）
- 各模块内容（Normal样式，带编号）
- 图片（居中，带Caption说明）
- 总结（Normal样式）

样式说明：
- 标题：Heading 2 样式
- 图片说明：Caption 样式
- 正文：Normal 样式
- 代码/公式：按需使用相应样式
```

## 示例

### 输入示例

```

作者：xxx

   - 创建了翅片式散热器三维模型

总结：散热器设计满足热设计要求，最高温度低于70℃阈值。
```

## 高级功能

### 批量处理

支持一次性生成多周周报：
```bash
python scripts/generate_docx.py --batch weekly_reports_config.json
```

### 模板定制

用户可提供自定义模板：
```bash
python scripts/generate_docx.py --template custom_template.docx --config config.json
```

### 图表自动编号

脚本自动为图片添加连续编号，支持插入新图后自动更新后续编号。

## 注意事项

1. **图片质量**：建议使用png格式，分辨率不低于300dpi
2. **文件路径**：使用绝对路径或相对于脚本的路径
3. **中文支持**：确保系统安装了中文字体（如宋体、微软雅黑）
4. **备份**：生成前建议备份已有周报，避免覆盖
